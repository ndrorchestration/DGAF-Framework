## Procedure 1 — Technology Identification

**Trigger:** Scheduled scouting cycle, OR Oracle horizon scan identifies technology inflection signal.

1. Survey technology frontier: research preprints, open-source ecosystems, startup activity, academic output
2. Identify technologies at TRL 1–4 (pre-mainstream)
3. For each identified technology, create a pipeline entry with:
   - Technology name and one-paragraph description
   - Source of identification
   - Preliminary TRL estimate
   - Initial strategic relevance note
4. Add to innovation pipeline in VANGUARD_MEMORY.md
5. Route signal to Oracle (temporal framing request)

---

## Procedure 2 — Technology Readiness Assessment

**Trigger:** Pipeline entry reaches evaluation threshold (strategic fit ≥ 0.5 preliminary).

1. Conduct full TRL assessment (1–9):
   - Cite specific evidence for each TRL claim
   - Document methodology used
2. Score strategic fit (0.0–1.0):
   - Alignment with current Quintet strategy
   - Complementarity with existing ecosystem capabilities
3. Estimate adoption timeline in collaboration with Oracle
4. Build risk profile stub
5. Submit risk profile to Sentinel-Phi for φ-bounded review
6. On RISK_FLAG: revise or escalate to Amethyst
7. Submit completed assessment to Prof Prodigy for adoption curve verification
8. Submit to Apogee evidence gate
9. On PASS: present to Quintet with recommended action
10. Update VANGUARD_MEMORY.md pipeline entry with assessment result

---

## Procedure 3 — First-Mover Window Calculation

**Trigger:** Technology assessment PASS with TRL ≤ 5 and strategic fit ≥ 0.7.

1. Identify current mainstream adoption threshold (S-curve inflection point estimate)
2. Estimate time to mainstream adoption (work with Oracle for temporal probability weighting)
3. Calculate first-mover window: from now to inflection point − lead-time required
4. Classify window: open (>12 months) / closing (6–12 months) / closed (<6 months)
5. Route window classification to Nova (disruption strategy) and Zenith (performance prep)
6. Flag to Sentinel-Phi if window requires accelerated risk review

---

## Procedure 4 — Pipeline Routing and Hygiene

**Trigger:** Quarterly pipeline review, OR pipeline exceeds 10 active entries.

1. Review all pipeline entries for staleness (no update in 30+ days)
2. Re-evaluate stale entries: still relevant? TRL changed?
3. Archive or close entries that no longer meet strategic fit threshold
4. Re-prioritize remaining entries by: window status × strategic fit × risk level
5. Present top-3 to Quintet for resource allocation decision
6. Update VANGUARD_MEMORY.md pipeline archive
