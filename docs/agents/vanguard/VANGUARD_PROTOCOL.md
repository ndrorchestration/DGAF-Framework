## Procedure 1 — Technology Identification

**Trigger:** Scheduled scan; Amethyst signal; Oracle horizon scan referral; external signal.

```
Step 1: Scan relevant signal sources
        — Research output, open-source activity, startup activity,
          academic publication, patent filings

Step 2: For each identified technology:
        — Assign TRL (1–9)
        — Assess strategic fit (high / medium / low) against
          current formation objectives
        — Estimate adoption timeline (near / mid / long horizon)

Step 3: Add to pipeline (if TRL 1–4)
        — Log in MEMORY.md pipeline table
        — Route signal to Oracle for temporal framing

Step 4: If TRL 5–6: initiate Procedure 2 (readiness assessment)
Step 5: If TRL 7–9: log as mainstream; remove from active pipeline
```

## Procedure 2 — Technology Readiness Assessment

**Trigger:** Pipeline item reaches TRL 3+; Oracle temporal framing received.

```
Step 1: Evaluate technology on four dimensions:
        — Maturity (TRL equivalent; cite evidence)
        — Strategic fit (alignment with formation objectives)
        — Risk profile (preliminary — full review by Sentinel-Phi)
        — Adoption timeline estimate (coordinate with Oracle)

Step 2: Request Prof Prodigy verification
        if adoption curve involves mathematical modeling

Step 3: Route full assessment to Sentinel-Phi for φ-bounded risk review
        — Wait for CLEAR or RISK_FLAG before routing to Nova

Step 4: On Sentinel-Phi CLEAR:
        — Route assessment to Nova (disruption strategy)
        — Route to Zenith if performance optimization application identified

Step 5: Submit to Apogee for final gate verification

Step 6: Update MEMORY.md pipeline and archive tables
```

## Procedure 3 — First-Mover Window Calculation

**Trigger:** Technology reaches TRL 3–4; Oracle scenario set references technology.

```
Step 1: Define emergence point (current TRL + signals)
Step 2: Model mainstream adoption threshold (TRL 7 equivalence)
Step 3: Calculate window duration:
        window = adoption_threshold_date - emergence_date
Step 4: Request Prof Prodigy verification of adoption curve model
Step 5: Route window estimate to Oracle for scenario probability weighting
Step 6: Flag to Amethyst if window < near-term horizon
        (urgent first-mover opportunity)
```

## Procedure 4 — Pipeline Management

**Trigger:** Session open; new technology added; assessment completed.

```
Step 1: Review all active pipeline items
Step 2: Update TRL estimates based on new signals
Step 3: Reprioritize pipeline:
        — Priority 1: TRL 3–4 + high strategic fit + short window
        — Priority 2: TRL 1–2 + high strategic fit
        — Priority 3: TRL 3–4 + medium strategic fit
Step 4: Remove completed or mainstream items
Step 5: Update MEMORY.md pipeline table
```
