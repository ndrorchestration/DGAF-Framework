## Modal Frequency Gating (Constraint Hierarchy Modeling)

**Accepted term:** Constraint hierarchy — a structured ordering of constraints by priority, where higher-tier constraints override lower-tier ones. Modal gating refers to controlling which operational mode (state) a system may enter based on current constraint conditions.

**Ecosystem coinage:** "Modal frequency gating" — DemiJoule's implementation of constraint hierarchy modeling using the formation's modal frequency framework (where each operational mode corresponds to a frequency band).

```
Modal frequency gate:
  Function:   Controls formation mode transitions
  Mechanism:  Each operational mode has a frequency threshold;
              DemiJoule gates mode entry by verifying constraint conditions
  Hierarchy:  Higher-priority constraints (sovereign/T3) block lower-priority
              mode activations regardless of task demand
  Accepted anchor: Constraint satisfaction / modal logic gating

Gate conditions:
  OPEN:   All constraints for requested mode satisfied
  HOLD:   Constraint pending resolution (formation waits)
  BLOCK:  Constraint violated — mode entry denied; escalate to Amethyst
```

## Energy Conservation (Resource Efficiency Optimization)

**Accepted term:** Resource efficiency — the principle of achieving maximum output with minimum resource expenditure. In computational and cognitive systems, this maps to avoiding redundant processing, unnecessary context switching, and cognitive overhead.

**Ecosystem coinage:** "Energy conservation" — DemiJoule models formation resource consumption using energy as the metaphor for cognitive/computational load.

```
Energy conservation principles:

  1. Focus management
     — DemiJoule tracks cognitive load across active formation agents
     — Flags redundant parallel reasoning chains for consolidation
     — Prevents "context thrashing" (excessive mode switching)

  2. Constraint efficiency
     — Applies the minimum necessary constraint set for each task
     — Avoids over-constraining formation (reduces paralysis)
     — Avoids under-constraining (prevents drift)

  3. Resource allocation balance
     — Works with Reciprocity to ensure balanced load distribution
     — Identifies agents operating outside their efficiency band
     — Flags to Amethyst for formation rebalancing

Accepted anchor: Resource efficiency / computational load management /
                 operations research optimization
```
